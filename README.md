bconv (Base Converter)
bconv 是一个专为嵌入式开发者和系统管理员设计的轻量级命令行进制转换工具。它不仅支持常见进制转换，还支持二进制位分组显示和位宽截断，方便调试寄存器值。

1. 核心功能设计
智能输入识别：自动识别 0x (Hex), 0b (Bin), 0 (Oct) 前缀，无需强制指定输入进制。

格式化输出：支持二进制按 4 位或 8 位分组显示 (e.g., 1100 1010)。

位宽控制：模拟嵌入式数据类型 (uint8, uint16, uint32)，自动处理溢出/截断。

简洁模式：既支持详细信息，也支持仅输出结果以便通过管道 (|) 传递给其他命令。

2. 命令格式 (Usage)
Bash

bconv [选项] <数值>
默认行为
如果不加任何“输出格式”选项，默认打印该数值的 十进制 (Dec)、十六进制 (Hex) 和 二进制 (Bin) 三种形式。

Bash

$ bconv 255
Dec: 255
Hex: 0xFF
Bin: 1111 1111
3. 详细参数 (Options)
输入控制
-i <base> : 强制指定输入进制 (Input Base)。如果省略，程序将尝试自动推断 (0x=Hex, 0b=Bin, 其他=Dec)。

输出控制 (互斥)
-x, --hex : 仅输出十六进制 (Hex)。

-d, --dec : 仅输出十进制 (Decimal)。

-b, --bin : 仅输出二进制 (Binary)。

-o <base> : 输出为任意指定进制 (2-36)。

格式化与修饰 (Modifiers)
-g, --group : 启用分组显示。二进制每 4 位加空格，十六进制每 4 位 (16bit) 加空格。

-w <bits> : 位宽限制 (Width)。支持 8, 16, 32, 64。

作用：模拟数据溢出。例如输入 0x1FF 且限制 -w 8，结果应为 0xFF。

-u, --upper : 强制大写 (主要用于 Hex, 如 0xFF vs 0xff)。

其他
-h, --help : 显示帮助信息。

-v, --version: 显示版本。

4. 使用场景示例 (Use Cases)
场景 A：快速查看寄存器值 (最常用)
我想看 0x3A 的二进制是多少。

Bash

$ bconv 0x3A
Dec: 58
Hex: 0x3A
Bin: 0011 1010
场景 B：配置 FPGA 引脚 (二进制分组)
我需要给一个 16 位的 GPIO 寄存器赋值，我想看 48879 对应的开关状态，且需要看得清楚。

Bash

# -b: 只看二进制
# -g: 开启分组(方便数格子)
$ bconv -b -g 48879
1011 1110 1110 1111
场景 C：模拟 uint8_t 溢出
我想知道 300 如果强行塞进 uint8_t 会变成什么。

Bash

# -w 8: 限制为 8 bit
$ bconv -d -w 8 300
44
场景 D：脚本管道流 (Pipeline)
我想把转换结果直接传给别的工具，不需要 Dec: 这种前缀。

Bash

# 仅仅输出结果字符串
$ bconv -x 255
0xFF
